# 企业多公司文件管理系统 - 需求规格说明书

## 1. 文档信息

### 1.1 文档版本
- 版本号：v1.0
- 创建日期：2026-04-05
- 最后更新：2026-04-05

### 1.2 文档目的
本文档旨在详细描述企业多公司文件管理系统的功能需求、非功能需求和约束条件，为系统设计、开发和测试提供依据。

## 2. 项目概述

### 2.1 项目名称
企业多公司文件管理系统

### 2.2 项目背景
随着企业规模的扩大，多公司协作场景日益增多，需要一个安全、高效的文件管理系统来实现多公司文件的隔离管理、权限控制和文件处理流程自动化。

### 2.3 运行环境
- **操作系统**：Windows / macOS
- **前端技术**：Vue 3 + Element Plus + Vite
- **后端技术**：Spring Boot 3.x + Spring Security + JWT
- **数据库**：H2（文件型数据库）
- **邮件服务**：QQ邮箱 SMTP
- **开发语言**：Java 17+ / JavaScript ES6+

## 3. 用户角色与权限

### 3.1 角色定义

#### 3.1.1 超级管理员（ADMIN）
- **权限范围**：最高权限，可管理所有公司、用户、文件
- **主要功能**：
  - 创建和管理公司
  - 创建、启用/禁用、删除用户
  - 重置用户密码
  - 选择是否发送邮件通知
  - 查看所有公司的文件
  - 上传文件（需选择目标公司）

#### 3.1.2 普通用户（USER）
- **权限范围**：仅管理本公司文件
- **主要功能**：
  - 文件上传（自动归属本公司）
  - 查看本公司文件
  - 查看上传记录
  - 下载处理后的文件

#### 3.1.3 禁用用户
- **状态**：无法登录系统
- **说明**：用户注册后默认为禁用状态，需管理员启用后方可使用

### 3.2 权限矩阵

| 功能模块 | 普通用户 | 超级管理员 |
|---------|---------|-----------|
| 登录系统 | 需启用后 | 随时 |
| 注册账号 | ✓ | ✓ |
| 注册后状态 | 禁用（默认） | 启用 |
| 查看本公司文件 | ✓ | ✓ |
| 查看所有公司文件 | ✗ | ✓ |
| 上传文件 | ✓ | ✓（需选公司） |
| 管理用户状态/密码 | ✗ | ✓ + 邮件可选 |
| 管理公司 | ✗ | ✓ |
| 删除用户/公司 | ✗ | ✓ |

## 4. 功能需求

### 4.1 基础功能

#### 4.1.1 系统登录
- **功能描述**：账号密码登录
- **输入项**：用户名、密码
- **处理逻辑**：
  - 验证用户名和密码
  - 检查用户状态（禁用用户无法登录）
  - 生成JWT Token
- **输出项**：登录成功/失败提示，跳转到首页

#### 4.1.2 公司注册
- **功能描述**：管理员创建新公司
- **输入项**：公司名称
- **处理逻辑**：
  - 检查公司名称是否已存在
  - 创建公司记录
- **输出项**：创建成功/失败提示

#### 4.1.3 用户注册
- **功能描述**：新用户注册账号
- **输入项**：账号、密码、所属公司、注册邮箱
- **处理逻辑**：
  - 验证公司是否存在（不存在则提示）
  - 检查用户名是否已存在
  - 创建用户记录（默认状态为禁用）
- **输出项**：注册成功提示（需等待管理员启用）

### 4.2 核心业务功能

#### 4.2.1 文件上传
- **功能描述**：上传文件到INPUT目录
- **上传方式**：点击选择 / 拖拽上传
- **权限控制**：
  - 普通用户：直接上传（自动归属本公司）
  - 管理员：必须选择目标公司后上传
- **文件重命名规则**：`公司名_原始文件名.后缀`
- **存储位置**：INPUT目录（dirA）
- **记录信息**：
  - 原始文件名
  - 新文件名
  - 上传时间
  - 上传人
  - 公司名
  - 文件状态

#### 4.2.2 上传记录
- **功能描述**：查看文件上传历史记录
- **权限控制**：
  - 普通用户：仅查看本公司记录
  - 管理员：查看所有公司记录
- **显示信息**：
  - 原始文件名
  - 新文件名
  - 上传时间
  - 上传人
  - 公司名
  - 文件状态

#### 4.2.3 INPUT目录查看
- **功能描述**：查看INPUT目录中的文件列表
- **权限控制**：
  - 普通用户：仅查看本公司文件
  - 管理员：查看所有文件
- **显示信息**：
  - 文件名
  - 文件大小
  - 最后修改时间

#### 4.2.4 OUTPUT目录查看与下载
- **功能描述**：查看外部程序处理后的PDF/JPG文件
- **目录结构**：`dirB / 公司名 / 文件`
- **权限控制**：
  - 普通用户：仅查看本公司文件
  - 管理员：查看所有公司文件
- **功能操作**：
  - 查看文件列表
  - 下载文件
- **显示信息**：
  - 文件名
  - 文件大小
  - 最后修改时间

### 4.3 管理员高级功能

#### 4.3.1 用户管理
- **功能描述**：管理所有用户
- **功能列表**：
  - 查看用户列表（账号、公司、状态、邮箱）
  - 启用/禁用用户
  - 重置用户密码（默认密码：qweasdzxc）
  - 删除用户
  - 选择是否发送邮件通知

#### 4.3.2 公司管理
- **功能描述**：管理所有公司
- **功能列表**：
  - 查看公司列表
  - 创建新公司
  - 删除公司（需确认公司下无用户）

#### 4.3.3 文件管理
- **功能描述**：管理所有文件
- **功能列表**：
  - 查看所有公司INPUT目录文件
  - 查看所有公司OUTPUT目录文件
  - 全权限下载

## 5. 非功能需求

### 5.1 易用性
- 支持拖拽上传文件
- 提供一键刷新功能
- 提供一键下载功能
- 界面简洁直观
- 操作提示清晰

### 5.2 安全性
- 公司文件严格隔离
- JWT Token认证
- 密码加密存储
- 权限严格控制
- SQL注入防护
- XSS攻击防护

### 5.3 灵活性
- 管理员可自由控制邮件发送
- 支持多种文件格式
- 可配置文件大小限制
- 可配置存储路径

### 5.4 数据持久化
- 数据库持久化存储
- 重启不丢失数据
- 文件持久化存储
- 支持数据备份

### 5.5 兼容性
- 100% 支持 Windows
- 100% 支持 macOS
- 支持主流浏览器（Chrome、Firefox、Edge、Safari）

### 5.6 性能要求
- 文件上传响应时间 < 5秒（10MB以内）
- 页面加载时间 < 3秒
- 支持并发用户数 > 50

## 6. 约束条件

### 6.1 技术约束
- 使用Spring Boot 3.x框架
- 使用Vue 3框架
- 使用H2文件型数据库
- 使用JWT进行身份认证

### 6.2 业务约束
- 用户必须归属一个公司
- 文件命名必须包含公司名
- 管理员上传文件必须选择公司
- 用户注册后默认禁用

### 6.3 环境约束
- 需要Java 17+运行环境
- 需要Node.js 16+运行环境
- 需要配置邮件服务器

## 7. 数据需求

### 7.1 数据实体

#### 7.1.1 用户（User）
- ID：唯一标识
- 用户名：登录账号
- 密码：加密存储
- 邮箱：用于接收通知
- 公司ID：所属公司
- 角色：ADMIN/USER
- 状态：ENABLED/DISABLED
- 创建时间
- 更新时间

#### 7.1.2 公司（Company）
- ID：唯一标识
- 公司名称：唯一
- 创建时间
- 更新时间

#### 7.1.3 文件记录（FileRecord）
- ID：唯一标识
- 原始文件名
- 新文件名
- 文件路径
- 文件大小
- 公司ID
- 上传人ID
- 上传时间
- 文件状态：UPLOADED/PROCESSING/COMPLETED/FAILED

### 7.2 数据关系
- 一个公司包含多个用户（1:N）
- 一个用户归属一个公司（N:1）
- 一个公司包含多个文件记录（1:N）
- 一个用户上传多个文件记录（1:N）
