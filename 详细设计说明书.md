## 2. 系统架构设计

### 2.1 总体架构

#### 2.1.1 架构模式
采用前后端分离的B/S架构，分为三层：
- **表现层**：Vue 3 + Element Plus
- **业务逻辑层**：Spring Boot 3.x
- **数据访问层**：Spring Data JPA + H2 Database

### 4.3 认证与授权

#### 4.3.1 JWT认证流程
```
1. 用户登录 → 服务器验证
2. 验证成功 → 生成JWT Token
3. 返回Token给客户端
4. 客户端存储Token（localStorage）
5. 后续请求携带Token（Authorization: Bearer <token>）
6. 服务器验证Token → 解析用户信息 → 授权访问
```

#### 4.3.2 权限控制
- **注解方式**：使用`@PreAuthorize`注解控制接口权限
- **角色判断**：`hasRole('ADMIN')` 或 `hasRole('MANAGER')` 或 `hasRole('USER')`
- **数据隔离**：在业务逻辑中根据用户角色过滤数据
